# 08 — 資安與隱私

> 個案評估資料屬敏感個人資料，須符合臺灣《個人資料保護法》（PDPA）。本文件依儲存策略分兩階段（見 [02_architecture.md](02_architecture.md)）：
>
> - **開發期（單機）**：資料只存在裝置的瀏覽器內（IndexedDB），不離開裝置 — 除非使用者主動**匯出**。重點：本地儲存保護與**匯出檔的處理**（見 8.7）。
> - **未來（資料上傳 PostgreSQL）**：個資進入雲端，認證、加密、存取隔離、合規（8.2–8.6）全面適用。
>
> 各節標示適用階段。

## 8.1 威脅面與原則

| 原則 | 說明 |
| --- | --- |
| 最小蒐集 | 只存必要欄位。開發期使用者輸入必填僅 `name`（`patientId` 為系統產生 UUID，見 [06_data_model.md](06_data_model.md) 6.2）；非必要個資不蒐集 |
| 去識別化優先 | 以代碼／ID 為主鍵，盡量降低可直接識別的個資；姓名等敏感欄位可加密或分離儲存 |
| 目的限定 | 資料僅用於評估與治療紀錄，不作他用 |
| 最小權限 | 治療師僅能存取自己（或所屬院所）的個案 |
| 預設安全 | 全程 HTTPS、儲存加密、預設關閉公開存取 |

## 8.2 認證與授權（上傳 PostgreSQL 後適用）

- **認證（Authentication）**：治療師帳號登入（Email／OAuth），以 session／JWT 維持登入；支援登出與逾時。開發期單機無帳號。
- **授權（Authorization）**：以擁有者／院所為界，限制個案與評估的存取範圍。
  - 採 Supabase 時可用 **Row-Level Security（RLS）** 在資料庫層強制；自建後端則於 API 層檢查。
- **角色**：初期單一「治療師」角色；後續可擴充院所管理者、唯讀等。

## 8.3 傳輸與儲存加密（上傳 PostgreSQL 後適用）

| 層面 | 要求 |
| --- | --- |
| 傳輸中 | 一律 **HTTPS／TLS**；不接受明文 |
| 儲存時 | 資料庫與物件儲存啟用 **at-rest 加密** |
| 敏感欄位 | 姓名等可識別欄位考慮**欄位級加密**或與識別碼分離儲存 |
| 金鑰 | 金鑰與祕密以環境變數／祕密管理服務保存，全大寫命名（見 [07_dev_conventions.md](07_dev_conventions.md)），不入版控 |

## 8.4 3D 資產與個案資料的存取分離（兩階段皆適用）

- **通用解剖模型（glTF）**：非個資，可公開經 CDN 快取，加速載入 — 開發期即如此。
- **個案／評估資料**：開發期只存本地，**永不經 CDN**；未來上雲後須經認證 API 存取，如需附件使用**簽章 URL（signed URL）**且短時效。

## 8.5 個資法（PDPA）合規

- **告知與同意**（兩階段皆適用）：蒐集個案資料前取得告知同意（蒐集目的、項目、利用範圍、期間、當事人權利）— 即使單機儲存，蒐集行為本身即適用。
- **當事人權利**（兩階段皆適用）：提供查詢、更正、刪除個案資料的機制；開發期＝本地編輯與刪除。
- **保存與刪除**：定義保存期限與到期／要求刪除的處理流程。
- **委外與資料所在地**（上傳後適用）：使用雲端供應商屬委外處理，須留意**資料儲存地區（data residency）**與合約（DPA）。
- **可攜**（開發期即具備）：評估紀錄可匯出（JSON，見 [06_data_model.md](06_data_model.md) 6.7），便於移轉與當事人請求。

## 8.6 稽核與紀錄（上傳 PostgreSQL 後適用）

- 對個案資料的存取與變更留下**稽核紀錄**（誰、何時、做了什麼）。
- 紀錄本身避免含不必要的個資；錯誤記錄勿輸出敏感欄位。

## 8.7 本地儲存與匯出檔（開發期的主要風險面）

開發期個案資料整份存於裝置瀏覽器（IndexedDB），且匯出功能會把個資寫成一般檔案：

- **裝置保護**：評估用裝置應啟用螢幕鎖／磁碟加密（BitLocker／FileVault／行動裝置預設加密）；瀏覽器使用者設定檔不與他人共用。
  - 裝置保護與匯出檔處理守則就地呈現於設定「關於」之可展開「資料安全指引」區塊。
- **瀏覽器資料**：「清除網站資料」會**抹除全部個案紀錄**；此外瀏覽器也**可能自動清除** — iOS Safari 對**未安裝**的網站有 7 天閒置清除政策、各瀏覽器於儲存壓力下可能逐出資料。對策：啟動時呼叫 `navigator.storage.persist()` 申請持久儲存（**Safari 17／iOS 17 起支援**，WebKit 依啟發式授予、安裝為主畫面 App 時最可能獲准；更舊版 iOS 不支援，准駁結果均應記錄以利除錯）、引導**安裝 PWA**（豁免 Safari 閒置清除並提高 persist() 獲准機率 — **iOS 上的主要防線**）、定期**手動匯出備份** — 匯出備份是最終保全手段。
- **匯出檔處理**：
  - 匯出檔為**明文 JSON 且含個資**，存放於受控位置，不上傳公開雲端硬碟、不經即時通訊軟體傳遞。
  - 轉移完成後刪除暫存的匯出檔。
  - 未來可選：匯出時以密碼加密封裝。
  - 匯出前以確認對話框提示「明文 JSON 含個資」與存放注意事項，確認才匯出、取消中止。
- **Service Worker**：不將含個資的內容快取到可被他人讀取之處；未來有帳號後，登出時清除本地敏感快取。
  - SW 預快取白名單僅含 js／css／html／svg／png／woff2，不含 json／db／sqlite／csv／idb 等個資副檔名（`src/pwa/precacheGlobs.ts` 單一真相，回歸測鎖定）。

## 8.8 開發期 vs 未來

| 項目 | 開發期（單機） | 未來（上傳 PostgreSQL） |
| --- | --- | --- |
| 帳號 | 無（單機） | 治療師登入 + 擁有者隔離 → 院所多角色 |
| 資料位置 | 裝置內 IndexedDB + 使用者自管匯出檔 | PostgreSQL（傳輸 + 儲存加密），本地轉快取 |
| 加密 | 依賴裝置層（螢幕鎖／磁碟加密）；可選匯出檔加密 | TLS + at-rest，敏感欄位級加密、金鑰輪替 |
| 合規 | 告知同意、本地查詢/更正/刪除、可攜（匯出） | 加上稽核、保存政策、DPA／資料所在地 |
| 去識別化 | 最小欄位、UUID 為主鍵 | 姓名與識別分離、假名化 |

## 8.9 開發者模式的安全邊界

開發者模式（見 [02_architecture.md](02_architecture.md) 2.11）免認證並輸出執行動作，須守住三條邊界：

- **建置期隔離**：開發者功能以建置旗標（`VITE_DEV_MODE`）編譯；正式建置剔除免認證路徑與記錄程式碼（`actionLogger` 為 no-op），**不存在**可於正式版開啟的執行期後門。
- **dev server 區網綁定隔離**：開發伺服器綁 `0.0.0.0`（全介面、區網可及，供實機 http 區網測試）**僅開發者模式生效**（`VITE_DEV_MODE`，見 [02_architecture.md](02_architecture.md) 2.11）；非開發者模式為 `localhost`（僅本機），不誤將服務曝露於區網。
- **環境隔離**（上傳 PostgreSQL 後適用）：開發者模式僅能連測試環境與測試資料，不得指向正式資料庫。
- **記錄去個資**：開發者動作記錄（dev → 瀏覽器 console）一律以識別碼指涉個案，不含姓名等可識別欄位；`actionLogger` 另對 detail 套用 `redactPii` 縱深防禦——遮蔽電子郵件／身分證字號／台灣手機號碼等結構化個資（UUID 主鍵不誤觸），姓名等非結構化個資仍依賴「僅識別碼」呼叫慣例。
